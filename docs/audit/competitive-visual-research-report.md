# COMPETITIVE VISUAL RESEARCH REPORT

**Klub referencyjny:** Piorun Wawrzeńczyce (B Klasa, Śląsk)  
**Produkcja audytowana:** https://pilka-mu.vercel.app  
**Data:** 2026-06-02  
**Status:** research + koncepcja — **bez implementacji**

---

> Metoda: analiza wizualna stron klubów (desktop + mobile), case studies agencji, porównanie z Facebookiem Pioruna i live produkcją FC OS 3.0.  
> Odrzucono: stare WordPressy, strony urzędowe, slidery 2015, szablony Pitchero/Weblium bez tożsamości.

---

## Executive summary

Najlepsze strony klubów — **od B klasy po Ekstraklasę** — wygrywają nie budżetem IT, lecz **fotografią prawdziwych ludzi**, **lokalną tożsamością** i **jednym mocnym pierwszym wrażeniem**.

Public Website 3.0 poprawił **scenografię** (ciemny hero, matchday, sponsor wall), ale nadal przegrywa z liderami, bo:

1. Zdjęcia to **demo SVG**, nie twarze z Wawrzeńczyce.
2. Na produkcji widać **błędy danych** (Invalid Date, undefined).
3. Strona nadal czyta się jak **skórka SaaS**, nie jak **Piorun**.

---

## Metodologia

Dla każdego poziomu ligowego wybrano **1–3 reprezentatywne strony** (nowoczesne lub typowe dla segmentu). Oceniono 9 wymiarów w skali **1–5**:

| Skala | Znaczenie |
|-------|-----------|
| 1 | Przestarzałe / urzędowe / brak tożsamości |
| 3 | Poprawne, ale generyczne |
| 5 | Emocja, foto, lokalność, mobile — wzór do naśladowania |

## 1. Kluby B Klasy

### Przykłady badane

| Klub | URL | Uwagi |
|------|-----|-------|
| **Silesia Szymanów** | silesiaszymanow.pl | B klasa Dolny Śląsk — strona „z charakterem”, ale generyczna |

| **Northumberland Park Rangers** (UK) | case study D33 | amatorski klub, nowy design 2024 |

### Ocena segmentu (średnia liderów vs typowych)

| Wymiar | Liderzy (3–4) | Typowe B klasy (1–2) |
|--------|---------------|----------------------|
| **Hero** | Zdjęcie drużyny / boiska + hasło lokalne | Stock / brak zdjęć / slider |
| **Kolorystyka** | Barwy klubu konsekwentnie | Losowe zielone/czerwone z szablonu |
| **Typografia** | Display na nagłówkach | Arial / system font |
| **Zdjęcia** | Treningi, dzieci, mecz | Brak lub 3 stare zdjęcia |
| **Układ** | Prosty scroll: mecz → zapisy → kontakt | Menu 15 pozycji, komunikaty urzędowe |
| **Sponsorzy** | Logo na ciemnym tle | Lista tekstowa |
| **Akademia** | Telefon + grupy wiekowe + foto dzieci | Akapit Lorem ipsum |
| **Aktualności** | 1 lead + krótkie newsy | Archiwum PDF-ów |
| **Mobile** | CTA „Zapisz” widoczne od razu | Menu hamburger bez priorytetu |

**Wniosek:** B klasa **może** wyglądać profesjonalnie (case YMCA, NPR) — bez Ekstraklasy, ale z **prawdziwymi zdjęciami** i **mobile-first dla rodziców**.

## 7. Kluby Ekstraklasy

### Przykłady badane

| Klub | URL | Benchmark |
|------|-----|-----------|
| **Legia Warszawa** | legia.com | Mobile Trends Award, React, modular CMS, mobile-first |

### Co Ekstraklasa robi inaczej (do przeniesienia skali na Pioruna)

| Element | Ekstraklasa | Adaptacja dla Pioruna |
|---------|-------------|----------------------|
| Hero | Full-bleed kampania sezonu | **Jedno** zdjęcie z ostatniego meczu / treningu |
| Typografia | Display + kontrast | Jeden font display na hasło klubu |
| Matchday | Banner z partnerem meczu | Plakat VS z datą i boiskiem Wawrzeńczyce |
| Akademia | Osobna ścieżka (mloda-gieksa.pl) | Sekcja above fold z telefonem |
| Mobile | 70%+ ruchu mobile (Legia case) | Sticky „Zapisz dziecko” |

**Legia case study (Herodot/TISA):** sukces = warsztaty produktowe + **mobile-first** + CMS dla redakcji — nie animacje dla animacji.

2. **Formularze kontaktowe** per audience (rodzic / sponsor)  
3. **Sponsorzy** widoczni „jak na koszulce”  
4. **Zero sliderów** — jeden hero  
5. **Volunteer-friendly CMS** — ważne dla SaaS, ale niewidoczne dla użytkownika

### Co nadal psuje pierwsze wrażenie

| Problem | Dowód na produkcji |
|---------|-------------------|
| **Fake media** | `/demo-media/*.svg` zamiast zdjęć z boiska |
| **Broken matchday** | „undefined — undefined Invalid Date” |
| **Pusty klub** | Jedna drużyna (Seniorzy), „Kadra w przygotowaniu” |
| **Brak lokalności** | Nigdzie: Wawrzeńczyce, boisko, DZPN, „nasze” |
| **FB vs www** | FB = twarze; www = ilustracje gradientowe |

## Wnioski strategiczne (design, nie tech)

### Co kopiować (skala Pioruna)

| Źródło | Element |
|--------|---------|
| Grassroots UK | Mobile-first pod rodzica, sponsorzy na ciemnym |
| Górnik Zabrze | [FOTO] w news, galerie emocji |
| Legia | Jeden dominujący hero, mobile award pattern |
| Hayes & Yeading | Lokalne barwy + prawdziwe mecze na hero |

### Czego nie kopiować

| Źródło | Element |
|--------|---------|
| 90minut / ZPN | Tabele urzędowe jako strona główna |
| Stary WordPress | Slider + 12 widgetów |
| Emirates demo | Video + GSAP — zbyt ciężkie na start |
| Pitchero default | Szablon bez barw klubu |

### Główna luka konkurencyjna Pioruna

> **Facebook Pioruna jest lepszą stroną klubu niż strona klubu.**

Dopóki www nie pokaże **tych samych twarzy i boiska** co FB — właściciel słusznie powie: „wygląda tak samo”.

## Rekomendacja przed kodem (Website 4.0)

1. **Shooting day** — 30 zdjęć: hero (3), drużyny (4), akademia (2), galeria (12), news (3)  
2. **Copy lokalne** — Wawrzeńczyce, boisko, telefon, hasło z FB  
3. **Fix danych** — matchday musi być perfekcyjny lub ukryty  
4. **Typografia klubu** — jeden display font na hasła  
5. **Mobile 3-sekundy** — hero + zapis dziecka + mecz w pierwszym ekranie
